This page contains a broad overview of my final project for Information Visualization.

Planned Tools

Since the data is contained in multiple CSV files, I plan on performing preprocessing in Python, using the Pandas library. Because I would like to create a network visualization, this involves transforming the data into either an adjacency matrix representation or sparse matrix columnar format. I will most likely be using the file containing a list of recipes by region. This file has in one column a country, and in the second column a list of ingredients used in that recipe. I will transform this data into a graph network representation in order to visualize similarity between recipes across parts of the world. Aside from using Python to transform the data, I do not foresee any other tools being needed for data preprocessing.

For implementation in D3 purposes, I plan on using the TopoJSON library to visualize regions of the world where these recipes are occurring.

Planned Techniques

Technique 1

This technique will be a visual network of shared ingredients across recipes for a certain region. In this case, nodes will be specific ingredients (column 2 in scirep-cuisines-detail/allr_recipes.txt) and edges will connect nodes if those ingredients appear in the same recipe (column 1 in scirep-cuisines-detail/allr_recipes.txt). I imagine this to be a highly connected network, and therefore a "backbone extraction" technique will most likely be needed to better visualize the connections between nodes. This technique essentially picks out the most important edges of a highly connected network to display. Thus, the weighted network will be more easily interpreted. The nodes will have the same structure in the visualization, but the edges will be shaded and styled according to how much weight is associated with that edge (how many recipes those two ingredients appear together in).

Technique 2

My second visualization will be a bar chart. Bars will originate from the x-axis, representing the highest occurring ingredient by region (unique values of column 2 in scirep-cuisines-detail/map.txt). The height of each bar will represent the percent of overall recipes for that region in which the ingredient occurred. Bars will be sorted alphabetically by the name of the ingredient which is being shown. This is implemented so that in the case where two regions have the same "most popular" ingredient, the user will be able to recognize this quickly.

Technique 3

The last technique implemented will be a map with "details on demand". A map of the world will be shown with countries outlined that I have recipe information for. When a user clicks on a country, a bar chart will be displayed showing the five most prevalent ingredients used in recipes for that country. This allows users to get a broad overview of which ingredients are common to certain countries rather than regions. Data for this technique will use both columns 1 and 2 in scirep-cuisines-detail/allr_recipes.txt.

Planned Interaction

It is my intent that the above mentioned map will be the first visual the user is introduced to. Upon interacting with this map, the user will be able to dive deeper into specific details and connections within the dataset.

I would like to implement mouseover highlighting on the map so that when the user hovers over a country they are able to receive information as to what region that country is in. This will help draw connections between country statistics and regional statistics. Details on demand will also be implemented on the map, so that when the user clicks on a country, another plot with the details discussed in technique 2 above will appear below the map. Interactivity within the network visualization will be limited as to not clutter the already very connected graph. When the user hovers over a node, a tooltip will appear with the name of the ingredient.
